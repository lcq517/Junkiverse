核心数据流
1
用户拍照 / 上传图片
前端调用相机或文件选择，图片在本地处理
2
前端 AI 抠图
使用浏览器端模型（@imgly/background-removal）本地抠图，零延迟，零成本
3
2D 自由拼贴
用户在 Canvas 上拖动、缩放、旋转部件，组合成角色
4
提交生成请求
前端导出拼贴图 + 部件信息，发送到后端
5
后端调用 AI API
拼接 prompt，调用 AI 绘画和文本生成接口
6
返回结果 + 保存作品
返回角色立绘和档案故事，可选保存到服务器
技术栈

前端

  • HTML5 + CSS3 + Vanilla JS（MVP 阶段）
  • Canvas API — 拼贴编辑器
  • @imgly/background-removal — 前端抠图
  • 响应式设计，移动端优先

后端

  • Spring Boot 3.x + Java 17
  • Spring Web — REST API
  • MySQL / H2 — 作品数据存储
  • RestTemplate / WebClient — AI API 调用
